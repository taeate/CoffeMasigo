8. commit
-> 트랜잭션이 성공적으로 완료되었음을 데이터베이스에 알려주는 명령어입니다. COMMIT 이후의 변경사항은 영구적으로 데이터베이스에 반영됩니다.

rollback

트랜잭션이 실패하거나 사용자에 의해 중단되었음을 데이터베이스에 알려주는 명령어입니다.
ROLLBACK 이후의 변경사항은 모두 취소되고, 트랜잭션 시작 전의 상태로 되돌아갑니다.


9. pk(기본키 Primary Key) : 중복불가, null불가, 테이블의 모든 데이터를 식별하는 컬럼

fk(외래키 Foreign Key) : 다른테이블의 PK 를 참조하는키, PK와 동일한 도메인을 갖게되며
			두테이블을 이어주며 참조하는 테이블의 무결성을 높여준다.

10. inner join, outer join
-> inner join 은 두 테이블의 공통 부분만을 가져옵니다
-> outer join 은 한 테이블의 정보와 그와 관련된 다른 테이블의 정보를 모두 가져옵니다.


11. DROP, DELETE, TRUNCATE의 차이
DROP은 테이블 자체를 데이터베이스에서 완전히 삭제합니다.
DELETE는 테이블의 행을 삭제하는 데 사용됩니다.(롤백가능)
TRUNCATE는 테이블의 모든 행을 빠르게 삭제합니다.
DELETE보다 더 빠르지만 조건을 지정할 수 없습니다. 모든 데이터가 삭제됩니다.


12. 자바의 특징
한 번 작성된 자바 프로그램은 다양한 플랫폼에서 변경 없이 실행할 수 있습니다.
자바는 객체 지향 프로그래밍 언어로, 상속, 다형성, 캡슐화 등의 객체 지향적 특징을 지원합니다.
메모리 관리를 자동으로 수행하는 가비지 컬렉션 기능을 제공합니다. 더 이상 사용되지 않는 객체를 자동으로 메모리에서 제거합니다.



13. JAVA 접근제어자
클래스, 변수, 메서드, 생성자의 접근 범위를 제한하기 위해 사용됩니다.
private -> 해당 멤버는 오직 그 멤버가 속한 클래스 내에서만 접근 가능합니다.
public -> 어떤 클래스에서든 해당 멤버에 접근할 수 있습니다.

14. Overriding VS Overloading
오버라이딩은 상속 관계에 있는 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것을 말합니다.
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}
장점: 기능의 재사용성을 높일 수 있습니다.

오버로딩: 같은 클래스 내에서 메서드 이름은 동일하지만 파라미터가 다른 경우.
장점: 메서드 이름을 기억하기 쉽게 만들어 코드의 가독성을 높입니다.


15. Static이란 무엇인가요
static 변수는 클래스에 대해 하나만 존재하며, 해당 클래스의 모든 인스턴스가 동일한 static 변수를 공유합니다. static 메서드는 인스턴스 변수나 인스턴스 메서드를 직접 사용할 수 없습니다.
